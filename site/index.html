نظرة عامة

FaceCore هو نظام كامل للتعرف على الوجوه يُست‑خدم في بيئات الإنتاج. يدمج مكتبة FastAPI لتقديم واجهة HTTP، و RetinaFace للكشف عن الوجوه، و ArcFace لإنشاء المتجهات المميّزة، مع تخزين المتجهات في FAISS للبحث القريب. يطبق نمط الهندسة النظيفة (Clean Architecture) بحيث تكون جميع الطبقات منفصلة عن بعضها.

الهندسة المعمارية

المخطط التالي يوضح تدفق البيانات عبر الطبقات.

        graph LR
        subgraph API[FastAPI Layer]
          A[GET /health] -->|تأكد من الحالة| B[Health Endpoint]
          C[POST /recognize] -->|صورة| D[Recognition Endpoint]
          E[POST /enroll] -->|صور + person_id| F[Enroll Endpoint]
        end
        subgraph Core[Core Logic]
          D --> G[RecognitionPipeline]
          F --> G
          G -->|detect| H[FaceDetector]
          G -->|embed| I[FaceEmbedder]
          G -->|match| J[Matcher]
          J --> K[FaissVectorStore]
        end
        subgraph Adapters[Adapters]
          H -->|RetinaFace| L[RetinaFaceDetector]
          I -->|ONNX / Torch| M[ArcFace (OnnxArcFace / TorchArcFace)]
          K -->|faiss‑cpu| N[FAISS Index]
        end
        API --> Core --> Adapters

المكوّنات الرئيسة

  • Detection: facecore/detection – واجهة FaceDetector وتنفيذ RetinaFaceDetector باستخدام insightface.
  • Embedding: facecore/embedding – واجهة FaceEmbedder مع OnnxArcFace (سرعة) و TorchArcFace (للتدريب).
  • Vector Store: facecore/recognition/vector_store.py – طبقة تخزين FAISS مع ملفات faces.faiss و faces_meta.json.
  • Matcher: facecore/recognition/matcher.py – يطابق المتجه مع قاعدة البيانات ويطبق عتبة match_threshold.
  • Pipeline: facecore/inference/pipeline.py – يربط الكشف، المحاذاة (facecore/preprocessing/alignment.py), التضمين والـ matching في خطوة واحدة.
  • API: facecore/api/main.py – نقاط النهاية /health, /recognize, /enroll, /verify. جميعها محمية بمفتاح API ومحدودة بمعدل طلبات.
  • Configuration: facecore/config.py – يستخدم pydantic‑settings قراءة المتغيّرات من .env. القيم في .env.example توضح كل إعداد.
  • Training: scripts/train.py + facecore/training – إعداد بيانات، تدريب نموذج ArcFace، حفظ checkpoint.

سير العمل (Recognition Flow)

  1. العميل يرسل صورة عبر POST /recognize.
  2. الـ FastAPI يمرّر الطلب إلى RecognitionPipeline.recognize.
  3. المكتشف RetinaFaceDetector يُعيد قائمة DetectedFace (مربعات + معالم).
  4. كل وجه يُمحاذَ عبر align_face لتوحيد الحجم 112×112.
  5. المحاذاة تُحوَّل إلى تمثيل Tensor ثم تُمرّر إلى FaceEmbedder (ONNX أو Torch) لتوليد متجه L2‑normalized.
  6. المتجه يُستَـخدم في Matcher.identify للبحث في FaissVectorStore، وتُسترجع أعلى تشابه لكل هوية.
  7. تُقارن النتيجة بالعتبة match_threshold؛ إذا كان أعلى من العتبة تُعيد هوية معروفة، وإلا تُعيد “unknown”.
  8. تُعيد API كائن JSON يحتوي على المربعات، الهوية، ومستوى التشابه.

التثبيت والبدء

# 1️⃣ استنساخ المستودع
git clone 
cd pro

# 2️⃣ إنشاء بيئة افتراضية (Python 3.12 يفضَّل)
py -3.12 -m venv .venv
.\.venv\Scripts\activate

# 3️⃣ تثبيت المتطلبات (ملف requirements.txt)
python -m pip install --upgrade pip
python -m pip install -r requirements.txt

# 4️⃣ إعداد المتغيّرات
copy .env.example .env
# عدّل .env إذا لزم الأمر (مسارات النموذج، المفاتيح، …)

# 5️⃣ تشغيل الخادم
python -m uvicorn facecore.api.main:app --host 0.0.0.0 --port 8000

بعد ذلك يمكن اختبار الواجهة عبر curl أو أداة مثل Postman.
